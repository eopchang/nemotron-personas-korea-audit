    Nemotron-Personas-Korea — Joint-Distribution & Dependency-Structure Audit¶

    English abstract of the audit. The detailed reports are written in Korean (the dataset's target audience). This document is a paper-style standalone summary for English-speaking readers, reviewers, and the synthetic-data evaluation community. For the canonical Korean version, see README.md. All numbers are reproducible from scripts/0[1-9]_*.py and 1[0-9]_*.py; numerical artifacts are mirrored in review/key_results.json.

    TL;DR¶

    NVIDIA's Nemotron-Personas-Korea (released April 2026) is a 1,000,000-row synthetic persona dataset for Korea, generated by a Probabilistic Graphical Model (PGM) seeded on KOSIS / Supreme Court / NHIS official statistics, with 26 demographic + 7 free-text persona fields. The data card publishes only marginal (univariate) qualitative comparisons; joint and conditional structure are not externally validated. This repository performs an independent quantitative audit in three phases — marginals (KOSIS comparison), bivariate joint structure (55 variable pairs), and conditional dependency skeleton (PC-style recovery + bias correction). Headline findings: the demographic chain (age → marital → family / age → education) reproduces Korean reality precisely, but housing_type is statistically decoupled from person attributes (a 1-person household and a 4-person household have nearly identical housing distributions), and military_status is essentially a deterministic function of occupation. The dataset is fit for its stated purpose (LLM training, synthetic-data diversity, bias mitigation, Sovereign-AI workflows) but unfit for person-level housing analysis, military-service flow analysis, or any task that requires the full joint distribution to track reality.

    1. Background¶

    The dataset card's stated intended use lists four scenarios: (i) LLM training, (ii) synthetic-data diversity expansion, (iii) data/model bias mitigation, (iv) Sovereign-AI model development. Within these scopes the dataset is well-positioned.

    However, two features create a real risk of out-of-scope adoption: 1. The framing "based on Korean official statistics" together with the 1M-row scale invites use as a proxy for census-style demographic data — for policy simulation, education materials, or downstream demographic research. 2. NVIDIA's published validation is limited to marginal distributions (gender ratio, province share, etc.). Because the data is generated by a PGM whose whole point is to control joint structure, the absence of joint-distribution validation leaves the most important property of the dataset unverified.

    This audit fills that gap.

    2. Methodology¶

    Phase 1 — Marginal fidelity (univariate)¶

    For each of 12 categorical variables, compute observed marginal distribution and compare with the closest available KOSIS reference using Total Variation Distance (TVD). χ² tests are deliberately not used: at N = 1M every micro-difference becomes "significant" (p < 0.001), so effect-size measures are the only meaningful tool.

    Five variables admit clean direct KOSIS comparison (sex, province, marital_status, education_level, housing_type); seven were excluded from primary comparison due to population-frame or taxonomy mismatch (documented per-variable in reports/tables/kosis_comparison.md and docs/FAQ.md).

    Housing required a special correction: KOSIS publishes per-household housing distribution but Nemotron is per-person. We constructed a per-person reference by re-weighting (1-person households 35.5% + multi-person averaging 2.21 members) — see scripts/20_housing_unit_correction.py.

    Phase 2 — Bivariate joint structure¶

    All 55 variable pairs (C(11, 2); country is constant and excluded) are evaluated with four complementary metrics: Cramér's V, Normalized Mutual Information (NMI), Theil's U (asymmetric), and TVD vs. independence. Each pair gets a three-panel visualization (joint heatmap, conditional row-stochastic, conditional col-stochastic). The full 55-pair atlas is in reports/figures/bivariate_all/; ten domain-curated pairs receive narrative treatment in Phase 2 §3.

    Phase 3 — Conditional dependency skeleton¶

    The structural workhorse. For all 55 pairs:

    1. Conditional Mutual Information (CMI) I(X; Y | Z) is computed for every Z (single-variable conditioning, |Z| = 1) and selected (best-mediator + 1) pairs (|Z| = 2).
    2. A PC-style skeleton classification labels each pair as:
    3. direct — survives all conditioning at threshold ε = 0.005 nats
    4. mediated — marginal MI ≥ ε but some Z removes it
    5. no_edge_marginal — marginal MI < ε
    6. ε sensitivity (§2.7): ε is varied over {0.0005, 0.001, 0.002, 0.005, 0.01, 0.02, 0.05}. Direct-edge count varies 32 → 13, but the headline conclusions (housing decoupling, demographic chain) are stable across the entire grid.
    7. Stratified permutation null + bootstrap CI (§2.5): for high-cardinality variables (occupation: 2,120 categories; district: 252; family_type: 39), plug-in MI carries a substantial upward bias ((k−1)(m−1)/(2N) ≈ 0.27 nats for occupation × district). To separate signal from this bias, we permute Y within Z (100 perms), giving a null distribution. The ratio observed / null_p95 classifies each edge into three tiers: robust (≥ 10), significant (2–10), bias-suspect (< 2). Result: only 12 of 23 "direct edges" survive as ratio ≥ 2; the other 11 (all involving occupation or district) are consistent with bias.
    8. Within-synthetic predictability check (§1.3, originally "decoupling probe"): an HGB classifier is trained on the synthetic data with baseline features vs. baseline + extras, and the cross-entropy difference (info_added) quantifies whether the joint structure carries information beyond a marginal-only baseline. NOT TSTR (we have no labels on real data).
    9. Leakage check (§1.4): the predictability check was re-run with train-only encoder, train-only cardinality cap, and 5-fold CV. All six conclusions hold (info_added shifts < 0.005 nats per case).
    10. External joint validation (§2.6): a partial KOSIS cross-tab comparison (age × marital × sex, age × sex × education, province × housing) confirms the demographic-chain finding (8 of 8 cells within ±4pp for the marriage table).

    Subsample stability (5 seeds × 200K rows): 52/55 pairs classify identically across seeds.

    3. Key Findings¶

    # Finding Verdict
    1 Province / district distribution matches KOSIS very closely. TVD = 0.005; all 17 provinces within ±0.24pp. ✅
    2 The demographic chain (age → marital → family, age → education) is quantitatively correct. 2020 census comparison: 8/8 cells of the age × marital × sex table within ±4pp; 25-34 4-year college 51.6%; 75-84 widowed 42.6% match KOSIS within rounding. ✅
    3 Gender × major separation matches Korean reality. Engineering 86% male, health & social services 28% male. ✅
    4 housing_type is statistically decoupled from person attributes. Single-person and 4-person households, young and elderly, all show ~62% apartment. Confirmed by the within-synthetic predictability check: info_added(person-attrs over district) = −0.008 nats (effectively 0), leakage-check-corrected to −0.0082 (5-fold SE < 0.02). ⚠️
    5 military_status is a deterministic function of occupation. U(military | occupation) = 1.000. The variable semantics, however, is "active-duty personnel status (career soldiers + conscripts combined)" — close to the U.S. "active duty" sense, not "currently serving conscription" — which correctly reflects Korean military rank composition (privates avg. 25, NCOs avg. 41, officers avg. 47, women 11%). ⚠️
    6 occupation's "unemployed/no occupation" category aggregates 36.7% of rows (homemakers + students + retirees + jobless). Fine-grained labor-market analysis is therefore limited, though the big picture (education → occupation, gender → occupation separation) works correctly. ⚠️

    Self-assessment caveat (★★★ in CLAIMS_LEDGER C31): of 23 "direct edges" produced by the PC-style step, only 12 are robust to high-cardinality bias correction (ratio ≥ 2). The other 11 — all involving occupation or district — should be treated as plausibly explained by plug-in MI bias.

    4. Practical Implications for Users¶

    Use case Recommendation
    LLM training / synthetic-data diversity / bias mitigation (NVIDIA's stated use) ✅ Fit for purpose
    Province- or district-level population simulation ✅ TVD = 0.005, very strong fidelity
    Demographic-chain studies (age × marital × family, age × education) ✅ Externally validated
    Labor-market gender separation (sex × field × occupation) ✅
    Active-duty rank composition (cross-section) ✅ Matches reality
    Fine-grained occupation/district joint analyses 🤔 Watch for high-cardinality bias artifacts
    Person-level housing analysis (single-person housing, youth housing poverty, etc.) ❌ Housing is decoupled from person attributes
    Conscription-flow / military-service-dynamics analysis ❌ Static cross-section only
    Time-series, life-course, or family-formation diversity studies ❌ Synthetic snapshot with deterministic constraints

    5. Limitations of the Audit¶

    1. Single dataset, single snapshot. No time series, no cross-version comparison.
    2. PC conditioning depth |Z| ≤ 2. Some "direct" edges could be mediated through |Z| ≥ 3 combinations. Mitigation: the headline housing and military findings are already clear at |Z| = 1.
    3. CMI threshold ε = 0.005 nats is a judgment call. Mitigation: ε sensitivity over a 100× range (§2.7) shows headline conclusions are ε-stable.
    4. No DAG direction recovery. The skeleton is undirected; orienting it requires extra assumptions (temporal order, domain knowledge) which we do not impose.
    5. High-cardinality bias affects ~half of the "direct" edges involving occupation or district. Quantified in §2.5 via permutation null; bias-corrected skeleton in reports/figures/skeleton_bias_corrected.png.
    6. Predictability check uses a single classifier family (HGB) and a single seed. Mitigation: 5-fold CV is reported (§1.4); multi-model robustness is planned in ROADMAP P8.
    7. External joint validation is partial. KOSIS API access for full cross-tabs is queued in ROADMAP P7 v2; current external validation uses cell-by-cell figures published in press releases.
    8. Phase 4 (free-text persona analysis: vocabulary diversity, stereotypes) is not yet performed (ROADMAP P9).

    7. For External Reviewers¶

    We actively solicit independent review. The repository is structured for low-friction review:

    • Single-file packet for LLM reviewers: review/REVIEW_PACKET.md (~35K tokens, no images).
    • Numbered, falsifiable claims: review/CLAIMS_LEDGER.md (36 claims with evidence-file links).
    • Pre-written review prompts: review/REVIEW_PROMPTS.md (Korean) / review/REVIEW_PROMPTS_EN.md (English). Six prompts: integrated review / falsification / methodology / statistics / domain / reproducibility.
    • Machine-readable numbers: review/key_results.json for mechanical verification.

    Please file findings as GitHub issues with label external-review.

    9. Acknowledgement & Scope¶

    This audit is not a critique of NVIDIA. The dataset is fit for its stated purposes; the findings here delineate the boundary between fit for stated use and fit for adjacent uses a reader might assume. We welcome corrections from NVIDIA, the original data card authors, or any reader who can demonstrate a flaw in the methodology or the numbers — see CONTRIBUTING.md.
